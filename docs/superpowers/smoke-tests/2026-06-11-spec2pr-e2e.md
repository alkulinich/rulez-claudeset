# spec2pr - manual end-to-end smoke test

Run once with REAL codex, Claude, and gh before first production use. The
stubbed test suite cannot verify external CLI contracts, `$superpowers:*`
prompt expansion under `codex exec`, Claude's JSON envelope shape, or gh auth.

## Setup

1. Create a scratch repo on GitHub (e.g. `spec2pr-smoke`), clone it, push a
   `main` with a README and `docs/superpowers/specs/` +
   `docs/superpowers/plans/`.
2. Write a toy spec at `docs/superpowers/specs/add-version-flag.md`:
   a small CLI script in the repo gains a `--version` flag printing `0.1.0`.
3. Confirm `codex exec 'say hi'`, `claude -p --output-format json 'say hi'`,
   and `gh auth status` work.

## Run

    bash ~/.claude/skills/rulez-claudeset/scripts/spec2pr.sh \
      <clone>/docs/superpowers/specs/add-version-flag.md

## Verify

- [ ] Status file `~/.rulez-claudeset/spec2pr/spec2pr-smoke-add-version-flag.status` shows
      every stage; final line is `SPEC2PR DONE pr=... worktree=...`; exit 0.
- [ ] Codex schema logs exist for spec review, plan, plan review,
      implementation, and any PR-review fixes.
- [ ] `$superpowers:writing-plans` and
      `$superpowers:subagent-driven-development` prompts expand correctly under
      `codex exec`.
- [ ] Claude review logs contain a valid `--output-format json` envelope with
      a `.result` review, and classifier logs parse to
      `{blockers_found, majors_found}`.
- [ ] Claude review and classify calls do not modify the worktree.
- [ ] Worktree exists with commits: import spec -> (review fixes) -> write
      plan -> (review fixes) -> implementation commits.
- [ ] PR open on GitHub with spec + plan + implementation; `--version` works
      in the worktree.
- [ ] PR has the best-effort spec2pr review summary comment.
- [ ] Logs in `~/.rulez-claudeset/spec2pr/spec2pr-smoke-add-version-flag/` contain prompt,
      stdout, stderr, JSON, review, classifier, and fix files for the relevant
      stages.
- [ ] Rerun the same command: completes fast, `pr exists` in status, no
      duplicate PR.
- [ ] Edit the source spec, rerun: `HALT preflight: source spec changed since
      import`.

This smoke test must pass once before first real use. Stub tests validate local
orchestration, but not real Codex schema behavior, Claude classifier parsing,
GitHub auth, or PR comment creation.
